# Manage a resource

Use the Resource Details page to view details and status for a resource and to perform life cycle operations on a resource.

## Before you begin

Role required: sn\_cmp.cloud\_service\_user

## Procedure

1.  Click **Resources** on the toolbar.

    The **Resources** page lists all resource types in the left pane.

    ![resource details](../image/resource-details.png)

2.  Navigate to the appropriate resource type to see the list of resources that you are interested in and then click a resource name to view specific details about that resource.

    ![Details pertaining to a specific resource.](../image/resourcename-details.png)

    The resource details screen contains the following information:

    -   Breadcrumb navigation
    -   All properties specific to the resource
    -   Activities associated with the resource. Click a tab to view:
        -   Change requests that are associated with the resource. See [Track a change request](cloudmgt-track-change-request.md).
        -   Incidents that were raised for the resource. See [Submit an incident for a stack](cloudmgt-submit-incident.md) and [Track an incident](cloudmgt-track-incident.md).
        -   Cloud events that happened to the resource. See [View cloud events](cloudmgt-view-cloud-events.md).
    -   Text box used to search the Cloud User Portal for stacks, resources based on types, stack requests, change requests, incidents, keys, and catalog items.
