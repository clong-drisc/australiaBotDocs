# Configuring Agent Client Collector Monitoring

Plan and configure your Agent Client Collector Monitoring implementation.

To configure Agent Client Collector Monitoring, follow these steps:

1.  [Configure a MID Server to work with Agent Client Collector Monitoring](../task/configure-mid-acc-monitoring.md).
2.  [Choose and configure metrics to monitor](../../event-management/task/configure-metric-monitors.md).
3.  [Configure the action level for anomaly detection](../../event-management/concept/configure-level-anomaly-detection.md).
4.  [Create a configuration settings rule](../../event-management/task/create-config-overriding-rule.md).
5.  [Enable metrics collection and evaluation](../task/acc-distributed-mid-cluster.md).

Depending on the checks and metrics you want to monitor, perform the following actions to enable full check activation and metrics collection:

1.  [Enable viewing HAProxy metrics](../task/configure-haproxy-metrics-stats-page.md).
2.  [Enable running of Varnish checks](../task/discover-varnish-ci.md).
3.  [Select the Azure policies to activate for metric collection](../task/select-policies-azure.md).
4.  [Create Azure policies for metric collection](../task/create-policies-azure.md).
