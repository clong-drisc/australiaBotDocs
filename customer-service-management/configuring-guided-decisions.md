# Configuring guidances and decision trees

Configure guidances and decision trees to resolve customer issues faster and provide standardized resolution.

A decision tree is a structured troubleshooting process for service agents to work through. For more information, see [Setting up Guided Decisions](setting-up-guided-decisions.md).

The configuration process consists of the tasks listed in the following table.

<table id="table_ikt_fvp_qlb"><thead><tr><th>

Step

</th><th>

Description

</th></tr></thead><tbody><tr><td>

[Create a guidance in the Core UI](../task/create-guidances.md)

</td><td>

Configure guidances to provide actions to your agents.

</td></tr><tr><td>

[Configuring decision trees in Decision Tree Builder](decision-trees-in-gdb.md)

</td><td>

Configure decision trees that agents can use to troubleshoot solutions for customer issues.You can create decision trees in Decision Tree Builder only.

</td></tr><tr><td>

[Editing decision trees in Core UI](decision-trees.md)

</td><td>

Edit the decision trees that you created using Core UI to improve or extend the troubleshooting process.Only editing of existing decision trees is supported in Core UI. You can create decision trees in Decision Tree Builder only.

</td></tr><tr><td>

[Add Guided Decisions to Recommended Actions](../task/add-guided-decisions-to-recommended-actions.md)

</td><td>

Add decision trees to recommended actions, so that agents can take relevant actions in a troubleshooting process based on a record context.

</td></tr><tr><td>

[Add Guided Decisions to playbooks](../task/add-guided-decision-playbook.md)

</td><td>

Add decision trees to playbooks, so that agents can provide resolution in a troubleshooting process for their cases.

[Add Guided Decisions to Service Portal](../task/add-guided-decisions-service-portal.md)

</td><td>
