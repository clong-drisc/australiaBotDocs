# Populate the stakeholder registry for demands

Add users to the stakeholder registry so that Next Experience for Demand Management can automatically populate the stakeholder list when demand users create a demand.

## Before you begin

Role required: pps\_admin

## Procedure

1.  Access the Stakeholder Register record form in one of the following ways.

    -   Navigate to **All** &gt; **Project Administration** &gt; **Settings** &gt; **Stakeholders**.
    -   Navigate to a demand playbook or the **Stakeholders** tab in **Details** page, and select **New**. For more information, see [Access demands](access-demand-workspace.md).
2.  On the Stakeholder Register record form, fill in the fields.

    For a description of the field values, see [Stakeholder register form](../reference/stakeholder-register-form-dw.md).

3.  Select **Submit**.

    **Note:** When adding a user to the stakeholder registry, the combination of the user and the assigned portfolio must be unique. However, there can be multiple records for the same user but with different portfolios.
