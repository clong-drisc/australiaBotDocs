# UX List form

Learn about the fields of the UX List form. Use this form to create additional lists in the Lists menu of Strategic Planning workspaces.

<table id="table_xk2_qdt_2wb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Title

</td><td>

Display name of the category in the workspace.

</td></tr><tr><td>

Application

</td><td>

Application scope. Ensure that this is set to **Portfolio Planning**.

</td></tr><tr><td>

Order

</td><td>

Order in which this category is displayed in the workspace.

</td></tr><tr><td>

Category

</td><td>

Category that this list would be displayed under, in the workspace.

</td></tr><tr><td>

Active

</td><td>

Option to enable the display of this category in the workspace.

</td></tr><tr><td>

Table

</td><td>

Source table for the list.

</td></tr><tr><td>

Configuration

</td><td>

Configuration that the category is referencing.

 By default, this field is set to **APW List Menu Configuration**.

</td></tr><tr><td>

Conditions

</td><td>

Conditions to filter the records of the source table.

</td></tr><tr><td>

Fixed query

</td><td>

Current fixed query conditions applied to the list.

</td></tr><tr><td>

Columns

</td><td>

Columns of the table that must be displayed in the list in the workspace.

</td></tr><tr><td>

Group by column

</td><td>

Attribute that would be used to group the list of records by.

</td></tr><tr><td>

View

</td><td>

View to determine which columns are displayed in the list in the workspace. If you select a view, the **Columns** field is disabled.
