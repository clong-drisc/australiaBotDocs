# MISP user information

You can use the MISP user information page to view all the associated users for the ServiceNow AI Platform MISP integration for Security Operations.

<table id="table_vh2_dsk_mqb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Source

</td><td>

User account's MISP source.

</td></tr><tr><td>

User ID

</td><td>

User ID that is assigned by the MISP server when a user account is created.

</td></tr><tr><td>

User Email

</td><td>

Email address of a user. Only organization administrators can see the email addresses of their own organization's users.

</td></tr><tr><td>

Organisation

</td><td>

Organization that a user belongs to.

</td></tr><tr><td>

Role Name

</td><td>

Role name that defines the user privileges that are attributed to a user. For more information about roles in MISP, see the [MISP documentation](https://www.circl.lu/doc/misp/GLOSSARY.html#roles).

</td></tr><tr><td>

Role ID

</td><td>

User role ID in MISP. For example, Perm Admin, Perm Audit, or Perm Tagger, and so on.

</td></tr><tr><td>

Permission

</td><td>

Permissions that a user has. The permissions that are available in MISP are as follows:-   Read Only
-   Manage Own Events
-   Manage Organization Events
-   Manage Publish Organisation Events

</td></tr><tr><td>

Sighting Permission

</td><td>

Permission that enables a user to perform sighting searches.

</td></tr><tr><td>

Tagger Permission

</td><td>

Permission that enables a user to do tagging.

</td></tr></tbody>
