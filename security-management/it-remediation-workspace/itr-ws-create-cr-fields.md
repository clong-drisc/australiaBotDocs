# Create change request form fields

The following table shows the fields and their description in the Create change request form.

<table id="table_yxp_4cq_rpb"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Change type

</td><td>

Normal – Normal is the default.

 Standard- two additional fields are displayed:

 Standard change category - Select a category from the list

 Standard change template – Select a template from the choice list to use for this change request

 Emergency - Critical priority

</td></tr><tr><td>

Add CIs to change request

</td><td>

Default is enabled. If enabled, the change request adds any configuration items \(CI\) that belong to active vulnerable items in this remediation effort are added to this change request.

</td></tr><tr><td>

Priority

</td><td>

High is the default. Select a priority from the list.

</td></tr><tr><td>

Assignment group

</td><td>

The assignment group is by default taken from the watch topic. Click the search icon, and from the modal choose another assignment group.

</td></tr><tr><td>

Planned end date

</td><td>

Click the calendar icon to select a planned end date.

</td></tr><tr><td>

Assigned to

</td><td>

The current user is by default assigned to the remediation task. Click the search icon, and from the modal choose another user.

</td></tr><tr><td>

Short description

</td><td>

The short description is taken from the remediation task. Click the text to edit it.

</td></tr><tr><td>

Description

</td><td>

The description is taken from the remediation task. Click the text to edit it.

</td></tr><tr><td>

Justification

</td><td>

The Justification taken from the remediation task. Click the text to edit it.

</td></tr><tr><td>

Implementation plan

</td><td>

Pre-populated with preferred solution information if it is available. View the solution or enter a justification plan.

</td></tr></tbody>
