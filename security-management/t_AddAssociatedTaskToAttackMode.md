# Add associated task to an attack mode/method

In addition to importing associated tasks \(such as changes and incidents\) as STIX data, you can add them to an attack mode/method manually.

## Before you begin

Role required: sn\_ti.admin

## Procedure

1.  Navigate to **All** &gt; **Threat Intelligence** &gt; **IoC Repository** &gt; **Attack Mode/Method**.

2.  Click the attack mode to which you want to add an associated task.

3.  Click the **Associated Tasks** related list.

4.  Click **Edit**.

5.  As needed, use the filters to locate the tasks you want to associate with the attack mode/method.

6.  Using the slushbucket, add the task to the **Associated Tasks** list.

7.  Click **Save**.
