# Software licenses in the legacy Software Asset Management plugin

Software licenses are based on models that are defined through the legacy Software Asset Management \(com.snc.software\_asset\_management\) plugin.

You can create the models to organize software licenses in any way that makes sense for your organization. Common methods of defining models and licenses include by department or by region.

Licenses can be associated with a contract. For more information, see the [Contract Management](../../contract-management/concept/c_ContractManagement.md) application.

## Licensing types

The following license types are available in the legacy Software Asset Management \(com.snc.software\_asset\_management\) plugin:

-   By CPU
    -   By CPU cores
    -   By number of CPUs
-   By number of points: Per installation - IBM PVU
-   By user
    -   Number of installs per user
    -   Per named user
-   By utilization
    -   Usage \(CPU\)
    -   Usage \(User\)
-   By workstation: Per workstation

|Model|Description|
|-----|-----------|
|Individual|Given to individuals.|
|Bulk allocated|Allocated to users via entitlements.|
|Bulk not allocated|Given to users with details, such as who has a license or where the license is located, not tracked.|
