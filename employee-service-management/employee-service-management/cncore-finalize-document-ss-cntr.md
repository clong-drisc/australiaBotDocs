# Work on a contract change request

As a contract fulfiller, review the change request, update the contract document, and send it back to the contract user.

## Before you begin

Role required: sn\_cm\_core.contract\_fulfiller

## Procedure

1.  Navigate to your workspace.

2.  Select the contract request for which the change request has been submitted.

    The details of the change request are included in the **Activity** stream and the notification email.

3.  If the contract request is not already assigned to you, select **Assign to me**.

4.  Select **Start work** to work on the change request.

5.  Select the Contract documents tab.

6.  Access and edit the contract document.

    1.  Select **Create revision**.

    2.  In the Create revision from drop-down list, select a storage type and upload the new document.

        -   **In the workspace**: Select **Attach File** and upload the document.
        -   **Activity stream**: In the **Document** field, select the document that is displayed based on attachments in the activity stream.

            The documents attached while requesting changes for a contract and the documents received through inbound emails are available in the Activity stream.

        -   **External Storage**- In the Document field, select the document. This option is displayed only if external storage is configured
8.  Get approval of the updated document from stakeholders.

    For more information, see [Initiate an ad hoc approval for a contract document revision](cmpro-initiate-approval-contract-request.md).

9.  Return the contract document to the requester.

    1.  Select **Send document**.

    2.  Select **Send document** on the confirmation screen.


## Result

The document is sent back to the user and the Contract status updates to Document ready to indicate that the contract document has addressed the change request.
