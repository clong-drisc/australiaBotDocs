# Configure dynamic clauses for contract templates

Enable dynamic customized content in contract templates through clauses that contain clause variations for preset conditions.

A clause is a collection of clause variations, which contain a block of content. For example, say only one section in a Non-disclosure Agreement contract document needs to be changed based on the geographic region of the signing legal entity. You would create multiple document templates with the identical content with the only difference being location-related content. You could then use the condition-based clause variations to create only one contract template that can generate documents for different locations.

Clauses are stored in a clause library, from which you can insert them in contract templates. You can create and add clause variations while creating a contract template or add them later.

## Benefits of clause variations

The benefits of using clause variations in document templates are:

-   Efficient use of a single contract template to produce contracts with different customized details.
-   Inclusion of a single clause variation in multiple contract templates.
-   Ability to define conditions and set the priority of execution so an appropriate content variation is picked up dynamically in the generated document.
