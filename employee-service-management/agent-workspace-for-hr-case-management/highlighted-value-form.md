# Highlighted value condition form

Use the Highlighted value condition form to create conditions to highlight a field.

<table id="table_xq2_ksp_11c"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Conditions

</td><td>

Conditions that trigger a field to be highlighted.

</td></tr><tr><td>

Color

</td><td>

Color of the highlighted field.

</td></tr><tr><td>

Application

</td><td>

Scope of applications this highlighting pertains to.

</td></tr><tr><td>

Order

</td><td>

The order in which to evaluate the conditions.When you have multiple conditions for highlighting the same field, they are evaluated in order from lowest to highest.

</td></tr><tr><td>

Hide Label

</td><td>

Toggle to hide label name.

</td></tr><tr><td>

Label

</td><td>

Name for the highlighted field that shows up when the condition is met.

</td></tr><tr><td>

Show Icon

</td><td>

Toggle to display the leading bullet when a field is highlighted.Example - ![Highlighted bullet](../../../administer/navigation-and-ui/image/bullet-highlighting.png)

</td></tr><tr><td>

Value Override

</td><td>

Text that replaces the field values from the table.If you don't supply a Value Override, Workspace highlights the field value from the table. If you supply a value, Workspace replaces the field value with your text and highlights that. Example - ![Override text](../../../administer/navigation-and-ui/image/override-text.png)

</td></tr></tbody>
