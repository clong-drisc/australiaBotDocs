# Move management key features and actions

As a Move manager, fulfill move requests efficiently by logically grouping them in to move projects using the Workplace Central Move management workspace.

## Move Management workspace

The Move management workspace is a one stop workspace where you can perform multiple actions on a move request raised by an employee or generated from a scenario deployment. The workspace enables you to manage your workload by creating move projects and monitor these requests by their expected start and due date.

The workspace has several key features such as move requests-related metrics, visual representation, detailed views, and also the option to create a move project. The Move management displays information in the following categories:

-   **Move management**: Displays metrics of your active move projects, move requests, and move tasks. It also provides a visual representation of the planned and unplanned move requests, states of requests and active projects.
-   **Create project**: Option to create a move project. For more information, refer to [Create a move project from existing move requests](../task/create-move-project.md).
-   **Move management**: Lists the details of the following:
    -   Details of all the move projects with their planned start and end date, their current state, and the assigned user.
    -   Details of all the move requests with their current state, the requested move date, the expected start, and the due date.

For detailed information about move projects, its features, and various states, refer to [Move project views, actions, and states](move-project-schedule-view.md).

## Move request metrics

As a move manager, you can view the metrics associated with the active projects that are assigned to you, the number of move requests and move tasks. You can select each tile to view more details in a list view. In the Move management metrics section, the following metrics are displayed:

-   **My active projects**: Number of active projects that you’re currently working on.
-   **Move requests**: Number of active move requests associated with the active move projects.
-   **Move tasks**: Number of active move tasks created to fulfill the active move requests.
-   **Planned vs unplanned requests**: Visual representation of planned and unplanned move requests in a Pie chart. Unplanned requests are those requests that don’t have an **Expected start** or **Due date** specified.
-   **Move requests \(pie chart\)**: Visual representation of move request states in a Pie chart.
-   **My active projects**: Visual representation of each move project and the number of cases associated to each project.

## Move project details

Under the Move management details section, you can view the details of all the move projects created in the organization. You can view details of the project such as the name of the project, current state, planned start, and end date and the assigned user. You can select a move project and work on it separately in a different tab.

At a time, the workspace displays up to five move projects. To view all the move projects, select **View all** at the end of the list.

-   When you select **View all**, the list is opened in a separate tab where you can view all the move projects created in your organization with their details.
-   You can also perform the following actions on the list:
    -   **Create**: Create a move project.
    -   **Delete**: Delete one or more move projects if they’re no longer required.

        **Warning:** Deleting a project will only remove the project and not delete the move requests associated to it. It unlinks the move requests from the deleted move project.

    -   **Refresh** the list.
    -   **Edit** the column.
    -   Add or remove filters.
    -   You can select a move project to view the details in a separate tab. On selection, the workplace move project is opened where you can view the project details and also edit any information such as the state if necessary.

## Move request details

Under the Move management details section, you can view the details of all the move requests created in the organization. You can view the details such as the request number, state, requested for user, request move date, expected start and due date, and the project to which the request is associated. You can select a move request to open it in a different tab.

At a time, the workspace displays up to five move requests. To view all the move requests, select **View all** at the end of the list.

-   When you select **View all**, the list is opened in a separate tab where you can view all the move requests created in your organization with their details.
-   You can also perform the following actions on the list:
    -   **Refresh** the list.
    -   **Edit** the column.
    -   Add or remove filters.
-   You can select a move request to view the details in a separate tab. When selected, the workplace move case record is opened where you can view the request details and also edit any information if necessary.

-   **[Move project views, actions, and states](move-project-schedule-view.md)**  
As a move manager, create, track, and work on move projects to fulfill multiple move requests at a time. The move project enables you to work using a schedule view, which is faster than opening individual move requests.
