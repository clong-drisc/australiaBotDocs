# Create a child case and a child task

As a case manager, to fulfill a case, you can create a child case and also a child task using the workspace.

## Before you begin

Role required: sn\_wsd\_case.manager

## Procedure

1.  Navigate to **All** &gt; **Workplace Central** &gt; **Workplace Central**.

    You can also open Workplace Central from the Employee Center directly. Navigate to **Workspaces** &gt; **Workplace Central**.

    The Workplace Analytics dashboard opens.

2.  On the left pane, select the **Case Management** icon \(![Case Management icon.](../images/casemgmt-icon.png)\).

    The Case Management landing page opens.

3.  To open the case to which you want to add a child case/task, do any of the following:

    -   Go to the **Overview** section.

        Select the relevant tile to open the list of cases.

    -   Go to the **All active cases** section.
4.  Select the case to which you want to add a child case/task.

    The case details are displayed in a separate tab. For more information about the view, the actions that you can perform and additional features, refer to the **Case details page** section in the [Case Management - Key features, Actions &amp; Case details](../reference/case-management-key-features-actions-case-details.md) topic.

5.  To create a child case, do the following:

    1.  On the case details page, select the **Child Cases** tab.

    2.  Select **New**.

    3.  On the form, fill in the fields.

        |Field|Description|
        |-----|-----------|
        |Workplace service|Select the workplace service of the case.|
        |Workplace location|Select the location for which you’re creating the child case.|
        |State|Select the state of the case. By default, the state is set as Draft.|
        |Assignment group|Select the assignment group to which you want to assign the case.|
        |Assigned to|Select the location for which you’re creating the child case.|
        |Requested for|Select the user for whom you are creating the case.|
        |Enable user acceptance|Select the option if you want to enable user acceptance for the case.|
        |Contact type|Specify the contact mode.|
        |Short description|Enter a short description about the case.|
        |Description|Enter a detailed description about the case.|
        |Additional comments|Add any comments regarding the case. The comments are visible to the requester.|
        |Work notes|Add work notes to the case. The notes are visible only to you.|
        |Time worked|Tracks the total time spent on a case or task. The timer starts at zero, tracks time until the record is saved or closed, and resumes when revisited. You can also manually adjust, pause, or resume the timer.|

    4.  To attach any file, select the Attachment icon \(![Attachment icon.](../images/casemgmt-attachementicon.png)\) on the side panel.

    5.  To configure a template for the case, select the Template icon \(![Template icon.](../images/casemgmt-templateicon.png)\).

    6.  Select **Save**.

        The child case is created.

6.  To create a child task, do the following:

    1.  On the case details page, select the **Child Tasks** tab.

    2.  Select **New**.

    3.  On the form, fill in the fields.

        |Field|Description|
        |-----|-----------|
        |State|Select the state of the task. By default, the state is set as Open.|
        |Workplace task type|Select the type of task.|
        |Assignment group|Select the assignment group to which you want to assign the task.|
        |Workplace location|Select the location for which you’re creating the child task.|
        |Assigned to|Select the location for which you’re creating the child task.|
        |Sign template|Select the signing template.|
        |Vendor|Select the vendor for the task.|
        |Workplace service activity|Select the Workplace service activity to implement the task.|
        |Short description|Enter a short description about the task.|

    4.  To attach any file, select the Attachment icon \(![Attachment icon.](../images/casemgmt-attachementicon.png)\) on the side panel.

    5.  To configure a template for the task, select the Template icon \(![Template icon.](../images/casemgmt-templateicon.png)\).

    6.  Select **Save**.

        The child task is created.

7.  After adding the child case/child task, select **Save** to save the changes.
