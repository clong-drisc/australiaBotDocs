# Work on a move project

After creating a move project, work on the project using the move scheduler. The move scheduler displays all the move requests in a schedule view \(calendar view\) and a list view. Track all the move requests that are grouped under the move project.

## Before you begin

Ensure that you have created a move project, and it’s active. For more information about creating a move project, see either of the following topics:

-   [Create a move project from existing move requests](create-move-project.md)
-   [Create a move project by uploading an Excel file](create-move-project-excel.md)

Ensure that you’re aware of the [Move project views, actions, and states](../reference/move-project-schedule-view.md).

Role required: sn\_wsd\_move.manager

## Procedure

1.  Navigate to **All** &gt; **Workplace Central** &gt; **Workplace Central**.

    You can also open Workplace Central from the Employee Center directly. Navigate to **Workspaces** &gt; **Workplace Central**.

2.  Select the Move management \(![Move management icon.](../images/move-mgmt-icon-central.png)\) module.

3.  In the **Move management** metrics section, select the **My active projects** tile.

4.  From the list view, select the project that you want to work on.

    The project opens in a separate move scheduler tab with a calendar view.

5.  View the users who raised the move requests by viewing the Move requests panel on the left.

    Scroll through the list to view all the users. The move requests are sorted based on their move request number.

6.  View the planed start and end dates on the calendar by scrolling horizontally on the calendar.

    **Important:** Move requests that are in the Unplanned state aren’t displayed with a bar. As a move manager, you must specify the planned start and end date to the move requests.

7.  View the previous week or next week by using the **Previous week/ next week** options \(![Previous next week icon.](../images/week-change-icon.png)\) placed on the top left of the calendar.

8.  Change the calendar view from week to month by using the **Week/Month** option \(![Week/Month icon.](../images/wsd-after-moving-rsv-evnt-planner-update-dialog.png)\).

    **Important:** You can’t move a move request bar in the **Month** view.

9.  View the keyboard shortcuts by selecting the Settings icon \(![Setting icon.](../../workplace-space-mapping/images/gear-icon.png)\).

10. View details of a move request by selecting the event/bar on the calendar.

    The move request details are displayed on the right in the Case details panel. To view more case details such as its child tasks, case SLAs, and more, select **Open case details**.

11. Update the move request details.

    1.  In the Case details panel, edit the fields that you want to change.

    2.  Select **Update**.

    3.  To view or update other case details such as the child cases, child tasks, approvers, case SLAs and more, select **Open case details**.

        The move request opens in a separate tab.

        1.  Edit the fields that you want to change.
        2.  Select **Save**.
12. Schedule a move request to another day.

    1.  Select the move request bar on the calendar.

    2.  Drag the request to the desired date.

        **Note:**

        -   You can drag the request only horizontally, that is, to another day on the calendar.
        -   You can’t drag the request to another user.
        -   You can’t drag a request if you are in the **Month** view.
    3.  Select **Save**.

13. Change the planned start and planned end dates of the move request.

    1.  Select the move request bar on the calendar.

    2.  In the **Case details** panel, edit the following fields:

    -   **Expected start**: Edit this field to change the expected start of the move request. That is, select the date from when you’re planning to start the move.
    -   **Due date**: Edit this field to change the due date of the move request. That is, select the date by when you want to close the move request.
14. Switch to a list view of the move requests by selecting the List option \(![List icon.](../images/list-options.png)\).

15. Remove a request or assign a move request to yourself.

    1.  Select the List option \(![List icon.](../images/list-options.png)\) to open the move requests in a list view.

    2.  To remove a request, do the following:

        1.  Select the move requests that you want to remove.
        2.  Select **Remove**.
    3.  To assign a move request to yourself, do the following:

        1.  Select the move requests that you want to assign to yourself.
        2.  Select **Edit**.
        3.  Select **Assign to me**.
16. Create move tasks for multiple move requests from the list view.

    1.  Select the requests that you want to create tasks for.

    2.  Select **Add tasks**.

    3.  On the pop-up form, select a Workplace task type, Workplace location, and Workplace service activity.

    4.  In the Short description field, enter information about the task.

    5.  Select an assignment group, then assign the task to a member of the assignment group.

    6.  Select the priority for the task, then select **Create**.

17. Add more move requests.

    1.  Select **Add new requests**.

        A separate tab opens with a list of move requests that aren’t assigned to any move project.

    2.  Select the move requests that you want to add.

    3.  Select **Add**.

18. To view the move project details, to change the state of the project, to assign the project to another move manager, to know the project's state transition, the actual start, and end dates and more, do the following:

    1.  Select the **Move project details** tab.

    2.  To change the state, edit the **State** field.

        Before changing the states, refer to [Move project views, actions, and states](../reference/move-project-schedule-view.md).

    3.  Assign the project to another move manager by changing the **Assigned to** field.

    4.  Add work notes.

        1.  Go to the **Compose** panel.
        2.  Enter your work notes.
        3.  Select **Post work notes\(private\)**.
        4.  Select **Save**.
    5.  Add or view attachments of the projects by selecting the Attachments icon \(![Attachments icon.](../images/attachment-icon.png).

    6.  Select **Save**.

19. Select **Save**.

20. Delete the move project:

    **Warning:** Deleting a project doesn’t delete or impact the associated move requests. After deletion, the move request will continue to exist in its current state without any project association.

    1.  Select More actions icon \(![More actions icon.](../images/more-actions-move-schedule.png)\).

    2.  Select **Delete**.

    3.  Select **OK**.


## Result

The changes are saved on the move project.
