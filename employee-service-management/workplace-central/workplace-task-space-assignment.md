# Workplace Task form - Space Assignment task

The Workplace Task form specifies details to assign a space to a neighborhood.

<table id="table_g5b_v4b_h3c"><thead><tr><th>

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Number

</td><td>

Task number for the workplace task.This field is auto-generated and can't be edited.

</td></tr><tr><td>

State

</td><td>

State of the workplace task.This field is automatically set to **Open** and can't be edited.

</td></tr><tr><td>

Workplace task type

</td><td>

Type of Workplace Task. This field is automatically set to **Assign spaces to neighborhood**.

</td></tr><tr><td>

Neighborhood

</td><td>

Neighborhood that the space must be assigned to.

</td></tr><tr><td>

Workplace task sub type

</td><td>

Sub-type of the space assignment task. You can select any of the following options based on your requirement.-   None: No sub type. The default value is **Append**.
-   Append: Assign the space to the list of existing spaces.
-   Remove all and replace: Remove existing spaces from the neighborhood and assign the new spaces.

</td></tr><tr><td>

Short description

</td><td>

Description for the workplace task.

</td></tr><tr><td>

Table

</td><td>

Table that is used to select spaces for the assignments. You can select either of the following options based on your requirement.-   Room \[sn\_wsd\_core\_room\]: Assign meeting rooms to the neighborhood.
-   Space \[sn\_wsd\_core\_space\]: Assign workspaces to the neighborhood.

</td></tr><tr><td>

Filter

</td><td>

Conditions used to filter spaces from the selected table. For more information about conditions, see [Condition builder](https://raw.githubusercontent.com/ServiceNow/ServiceNowDocs/australia/markdown/platform-user-interface/c_ConditionBuilder.md).

</td></tr></tbody>
