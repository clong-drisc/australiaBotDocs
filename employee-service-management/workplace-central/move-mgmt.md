# Move Management

Monitor multiple workplace moves at a time seamlessly using the Move Management workspace in the Workplace Central. Create move projects to track the progress of move requests.

The Move management workspace enables you to schedule moves as per their planned date. You can create a move project and group move requests using different filter conditions, such as, workplace location, campus, space type or more.

![Move scheduler view of a move project.](../images/move-scheduler.png "Move scheduler view of a move project")

A move project enables you to work on requests in a schedule view, which is referred to as the Move scheduler view. The Move schedule view provides a visual representation of move requests in a schedule view similar to the schedule view of the Workplace Reservation Management Reservation portal. The requests are displayed with their current state, the date on which they’re scheduled to move and further details. This way you can fulfill move requests as per the requested date in the Workplace Move Management. For more information about the Move scheduler, refer to [Move project views, actions, and states](../reference/move-project-schedule-view.md).

**Important:** The Move management workspace enables you to only view and schedule move requests across different dates. To fulfill a move request, you must use the Workplace Move Management.

A Move project enables you to logically group move requests that match conditions specified by you. You can view all the move requests that matched the conditions under a single move project.

For example, when there are around 100 move requests currently requested by employees, it becomes a hard task to prioritize the fulfilling of move requests. In such cases, you can do the following by using the Move Management workspace in the Workplace Central:

1.  Create a move project and specify condition filters to group move requests that match the condition. You can use conditions such as workplace location, space type, campus, requested move date, assignment group or more.
2.  In the project, work on the requests that matched the condition.
3.  View details of the move requests in the project in a calendar view. The request is displayed as a bar spread across the expected start and the due date.
4.  View the number of days available for a request to be fulfilled, the start and end date of the request, the assigned user to the request and more.
5.  Schedule the requests to another date by dragging across the calendar if necessary.

If you’re working on multiple move projects, you can monitor various metrics related to move requests, such as, the number of active projects, move tasks, number of planned and unplanned requests using the Move Management workspace.

As a move manager, using the Workplace Central Move Management workspace, you can perform the following:

-   Create a move project to track multiple move requests at a time. Specify conditions such as building, assignment group, space type and more based on which the move requests must be grouped.
-   Track the number of active move projects created that are created by you.
-   View the number of move requests associated with the move projects.
-   Track the number of workplace move tasks created to implement the move requests.
-   View the number of planned and unplanned move requests currently created in the application. You can know which requests you can actively work on
-   Visualize the current states of move requests. Track which requests are complete and which ones aren’t. Based on the state, you can work on any requests that requires attention.
-   You can also visualize each move project that you’re working on. Each project is displayed based on the number of move cases assigned to the move project.

For more information about the Move management workspace, refer to [Move management key features and actions](../reference/move-mgmt-views-states-actions.md).
