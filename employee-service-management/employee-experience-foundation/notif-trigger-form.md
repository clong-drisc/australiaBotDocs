# Trigger conditions form

The field descriptions provided here are essential to create the trigger conditions for your portal notification messages.

Field

</th><th>

Description

</th></tr></thead><tbody><tr><td>

Name

</td><td>

Given name for the trigger condition record.

</td></tr><tr><td>

Category

</td><td>

Category of notification for which the trigger is set.

</td></tr><tr><td>

Application

</td><td>

Scope of the application where the trigger condition record resides.

</td></tr><tr><td>

Active

</td><td>

Option to enable the trigger conditions set.

</td></tr><tr><td>

When to send

</td><td>

Trigger: Type of trigger that can be set.-   Record Change: Triggers a notification message each time there’s a change or update of records in the notifications table.
-   Event: Triggers a notification message when a particular event type is created.

</td></tr><tr><td>

When to send

</td><td>

Inserted: Option to enable notification triggering on the insertion of a new record.

</td></tr><tr><td>

When to send

</td><td>

Updated: Option to enable notification triggering on updating a record.

</td></tr><tr><td>

When to send

</td><td>

Table: Record table selected for which the trigger is set.

</td></tr><tr><td>

When to send

</td><td>

Conditions: Conditions to set off the trigger.-   Add Filter Condition: Field where the trigger conditions are added.
-   Add OR Clause: Field to add an optional clause for the trigger condition.

This field appears only when a record table is selected from the **Table** field.


</td></tr><tr><td>

When to send

</td><td>

Advanced condition: Advanced condition script to send a notification based on the current email address, change in the field values, or the system properties.

</td></tr><tr><td>

</td><td>

Users: Admin users to be added to the trigger conditions record.

</td></tr><tr><td>

Who will receive

Recipients listed in fields: Viewers to be added to the trigger conditions record.

Who will receive
