# Configure Journey designer features

Administrators configure Journey designer for the needs of their organization.

## Journey designer administration console apps and configuration

The Journey designer administration console consolidates Journey Accelerator, Journey designer, and Lifecycle Events configuration needs in a single intuitive page. Administrators can see which applications are already installed and other applications that are available for installation.

-   **Design a journey**

    Create and configure Lifecycle Events and Journey Accelerator plans to build and maintain journeys.

-   **Enhance the journey**

    Manage the features you already have installed and see what's available to install and enhance your journeys.


## Journey designer console helpful resources

Administrators new to Journey designer can use the resources available on the admin console page to learn about the features and workflow for creating journeys.

-   **Introduction**

    View a video that shows you the basic flow for creating and maintaining journeys. This video offers and detailed explanation about the relationship between Journey designer, Journey Accelerator, and Lifecycle Events.

-   **Getting started**

    Learn more about journeys, Lifecycle Events, and Journey Accelerator. This information helps you to understand everything involved in creating journeys.

-   **Helpful resources**

    Access the Journey designer guided setup or detailed documentation from the helpful resources listed in the console.


## Basic configuration process overview

The configuration process depends on the features and integrations used in your organization. Here are the basic process steps for configuring Journey designer.

-   **Step 1: Create and manage journey configurations**
    -   [Create and configure Journey designer journeys](../task/jny-dsgnr-journey-configuration.md)

        Journey designer configurations are used as templates for creating journeys. Administrators create configurations that are used to trigger journeys.

    -   [Manage Journey designer user roles](../task/jny-dsgnr-manage-users-roles.md)

        Your roles control access for creating and modifying different features in Journey designer.

-   **Step 2: Configure the Journey**

    Create a Journey type to classify the Journey configuration and associate the journey with a Lifecycle Events, Journey Accelerator, or both. [Create and manage Journey types](../task/jny-create-manage-journey-types.md)

    -   Lifecycle Events, see [Configure a lifecycle event](../task/configure-hr-lifecycle-event-type.md)

        **Note:** A journey that contains a life-cycle event must have an associated HR Service with the **Journey** fulfillment type. Either update an existing HR Service case or create a one. See [Configure an HR service for a journey](../task/configure-hr-service-for-journey.md)

    -   Journey Accelerator, see [Create a Journey Accelerator plan type](../task/jny-create-manage-plans.md)
-   **Step 3: Configure Journey permissions**

    [Update users' permissions for journeys](../task/jny-dsgnr-update-user-permissions.md)

-   **Step 4: Create and manage Journey designer reports and dashboards**

    [Journey designer dashboards and reports](../reference/jny-report-dashboards.md)


-   **[Create and configure Journey designer journeys](../task/jny-dsgnr-journey-configuration.md)**  
Create a journey and embed the necessary components from Lifecycle Events and Journey Accelerator.
-   **[Manage Journey designer user roles](../task/jny-dsgnr-manage-users-roles.md)**  
Manage Journey designer users and the roles that they have.
-   **[Manage Journey designer bulk updates for tasks](jny-dsgnr-bulk-update-admnstrtn.md)**  
Journey designer admins can configure which task types are available for bulk update and view information about tasks that were updated in bulk.
-   **[Creating and managing quick links for journeys](jny-dsgnr-quick-links-why-who.md)**  
Quick links give employees quick access to varied resources directly from a journey. Administrators, journey owners, and mentors are able to add linked resources to journeys.
-   **[Update users' permissions for journeys](../task/jny-dsgnr-update-user-permissions.md)**  
Users in a journey have permissions that enable different levels of access to a journey. Administrators modify user permissions for different types of journeys.
-   **[Create and configure Journey designer headers](../task/jny-dsgnr-header-configuration.md)**  
Manage journey headers or customize journey headers for different journey types based on user roles.
-   **[Create and manage Journey types](../task/jny-create-manage-journey-types.md)**  
Create plan types to manage the top-level classification for stages and tasks for Journey types, as well as the header views for users.
-   **[Create and manage plan types](../task/jny-create-manage-plans.md)**  
Create plan types to manage the top-level classification for stages and tasks in a Journey designer plan.
-   **[Create and manage plan configurations in Journey designer](../task/jny-create-manage-plan-configuration.md)**  
Create and manage plan configurations that are based on a plan type. Plan configurations are customized for specific employee audiences.
-   **[Setup the Journey designer plan configuration review and approval process](jny-dsgnr-plan-config-admin.md)**  
ServiceNow administrators configure the Journey designer plan configuration feature and process.
-   **[Enable the Now Assist Journey generation skill](../task/enable-jny-gen.md)**  
Enable the Journey generation skill in Now Assist for HRSD using the Now Assist Admin console.
-   **[Enable team tasks](../task/enable-team-tasks.md)**  
Enable team tasks to allow managers to add tasks to the Lifecycle Events activity sets that appear in the Skills widget on the Journey detail page. Enabling this feature gives managers and AI agents the flexibility to tailor journeys to individual employee needs.
-   **[Create and manage stages](../task/jny-create-manage-stages.md)**  
Create and manage stages that are contained inside a plan. Stages contain tasks or to-do lists for managers, mentors, or employees to complete.
-   **[Add stage configurations](../task/jny-create-stage-configuration.md)**  
Stages group tasks or to-do lists for managers, mentors, or employees as part of a plan configuration. You must set the stage order and associate the stage with a plan configuration.
-   **[Create a satisfaction score action](../task/jny-create-satisfaction-score-action.md)**  
A satisfaction score action triggers a widget to appear to the employee, containing either a URL or request to schedule a meeting, when the specific condition is met.
-   **[Creating and managing audience-specific templates](jny-creating-audience-specific-plans.md)**  
Administrators can develop Journey Accelerator templates for customized plans with stages that include user tasks for different audiences.
-   **[Recommended learning in journey configurations](jny-dsgnr-ml-learning-recom.md)**  
Administrators add relevant learning recommendations to a journey configuration.
-   **[Configure Journeys for Now Mobile](../task/configure-journeys-now-mobile.md)**  
Configure Journey designer in Now Mobile to enable managers, mentors, and employees to interact with their journeys from their mobile devices.
-   **[Initiate a learning task from a lifecycle event](../task/ln-task-pst.md)**  
When a lifecycle event is triggered, a learning task is created and assigned to the subject person of the case. The task is displayed in the To-dos page in Employee Center.
-   **[Employee Center \(EC\) homepage journey tabs widget](jny-dsgnr-ec-hp-widget.md)**  
The Service Portal Designer contains a library of widgets to add to a service portal. The **EC homepage journey tabs widget** displays key details related to users' journeys.
