# Journey designer reference

Reference topics provide additional information about the lists and forms that you use to configure and use Journey designer..

-   **[Task template fields](jny-task-template.md)**  
To make a task template, fill in the task template fields.
-   **[Journey designer Journey Configuration fields reference](jny-dsgnr-jrny-config-fields.md)**  
A journey configuration requires several fields to be completed or selected.
-   **[Journey designer Model task fields](jny-model-task-fields.md)**  
Managers create to-do tasks that are saved as templates. The templates are stored as a Journey designer Model Task.
-   **[Journey designer version 2 plan configuration fields](jny-dsgnr-v2-plan-config.md)**  
These fields are for Journey designer version 2 plan configurations.
-   **[Journey designer header configuration fields reference](jny-dsgnr-header-config-fields-ref.md)**  
Journey designer headers can be configured for different journeys so that the different users can have a unique view for their role in the journey.
-   **[Journey designer user permissions reference](jny-dsgnr-user-permission-ref.md)**  
Administrators can configure the user permissions for different Journey designer journey configurations.
-   **[Journey designer journey template reference](jny-dsgnr-plncnfg-info.md)**  
The journey template page has information that is useful to the plan configuration owner.
-   **[Components installed with Journey designer](components-installed-with-jny-dsgnr.md)**  
Several types of components install with the activation of the Journey designer application, including user roles and tables.
