# Create and configure Journey designer headers

Manage journey headers or customize journey headers for different journey types based on user roles.

## Before you begin

-   The **Application scope** must be set to Journey designer.
-   Role required: sn\_jny.admin

## About this task

There are default heading configurations for these personas:

-   **Employee**

    The person for who the journey is created.

-   **Journey owner**

    The creator of the journey. Owners are often managers for the employees on a journey.

-   **Mentor**

    A person who helps employees to complete journeys.


**Note:** A **Heading Configuration** must first be created to add heading configuration fields to.

## Procedure

1.  Navigate to **All** &gt; **Journey designer** &gt; **Manage Heading Configurations**.

2.  Select **New**, or open an existing Heading Configuration.

    Create header configurations for each persona included in a journey.

    -   Employee: The user the journey is created for.
    -   Mentor: A user that helps the employee to complete a journey.
    -   Owner: The user who created a journey. Journey owners are employee managers or mentors.
3.  Open a header configuration to modify or add fields to the configuration.

    **Note:** A header configuration must exist before **Heading Fields** can be added or changed.

    1.  Select **New**, or open an existing heading field.

        For more information see, [Journey designer header configuration fields reference](../reference/jny-dsgnr-header-config-fields-ref.md).

    2.  Fill out the **Heading Field** form, and select **Submit** or **Update**.
