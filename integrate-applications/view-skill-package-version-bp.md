# View skill package versions in a bot process

View a list of skill package versions used in a bot process in RPA Hub to make sure that you’re using the correct version.

## Before you begin

You can only see skill package versions in the Skill Overrides tab when the attended or unattended packages have the skill package associated to them.

For upgrading customers, to view the existing skill package versions in the Skill Overrides tab of a bot process, publish the attended or unattended package again and associate the package to the bot process.

Role required: sn\_rpa\_fdn.rpa\_release\_manager, sn\_rpa\_fdn.rpa\_support\_user, sn\_rpa\_fdn.rpa\_developer, sn\_rpa\_fdn.rpa\_business\_user, or sn\_rpa\_fdn.rpa\_admin

## Procedure

1.  Navigate to **All** &gt; **Robotic Process Automation** &gt; **RPA Hub Workspace**.

2.  Select the list icon \(![List icon.](../image/rpahublist-icon.png)\).

3.  On the **Lists** tab, under **Build**, select **Bot Process**.

4.  Open a bot process that you want to view a skill package version for.

5.  On the **Skill Overrides** tab, view a list of skill package versions associated to your bot process.
