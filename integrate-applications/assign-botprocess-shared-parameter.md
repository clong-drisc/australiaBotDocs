# Assign a bot process to a shared parameter in RPA Hub

Assign a bot process to a shared parameter in RPA Hub so that it can interact with this shared parameter. You can assign more than one bot process to a shared parameter.

## Before you begin

Create a bot process. For more information, see [Configuring a bot process record in RPA Hub](../concept/create-botprocess.md).

Create a shared parameter. For more information, see [Create a shared parameter in RPA Hub](create-shared-parameter.md).

Role required: sn\_rpa\_fdn.rpa\_release\_manager or sn\_rpa\_fdn.rpa\_admin

## Procedure

1.  Navigate to **All** &gt; **Robotic Process Automation** &gt; **RPA Hub Workspace**.

2.  Select the list icon \(![List icon.](../image/rpahublist-icon.png)\).

3.  On the **Lists** tab, under **Build**, select **Shared Parameters**.

4.  Open the shared parameter that you want to assign a bot process to.

5.  On the **Bot Process Configuration** tab, select **Add**.

6.  On the **Available Bot Processes** tab, select the required bot process.

7.  Select **Add**.

8.  Select **Save**.
